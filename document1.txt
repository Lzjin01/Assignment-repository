버전 관리 시스템(VCS) 
파일의 변화된 내용(변경, 추가, 삭제 등)을 시간 순서에 따라 시록한 후,
필요시 특정 시점의 버전으로 다시 불러와 사용할 수 있는 시스템.
대표적인 버전 관리 시스템 : Git, SVN

버전 관리 시스템의 종류
1. 로컬 버전 관리 시스템
	디렉터리로 파일을 복사하여 여러 개의 버전을 관리하는 방식
	단점 : 해당 파일이 실수로 삭제되거나 잘못 수정한 경우 되돌릴 방법이 없다
2. 중앙 집중식 버전 관리 시스템
	중앙에 파일을 관리하는 별도의 서버가 존재, 서버에 파일 버전을 기록/관리하는 데이터베이스가 존재
	장점 : 어떤 파일을 누가, 언제, 어떻게 수정했는지 쉽게 추적/관리가 쉬움
	단점 : 중앙 서버에서 문제가 발생하면 모든 사용자는 어떠한 작업도 수행할 수 없음
3. 분산 버전 관리 시스템
	중앙 서버에만 존재하는 파일 버전 관리/제어하는 데이터베이스가 모든 클라이언트에 복제됨
	클라이언트는 파일의 마지막 스냅샷을 다운받아서 사용하는 것이 아니라 저장소 전체를 복제하여 사용
